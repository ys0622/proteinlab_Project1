# Shake Template Guide

## 샘플 후보

카테고리 기준에 맞는 1차 샘플 후보는 아래 3개가 적절합니다.

- `플라이밀 프로틴쉐이크 초코`
- `랩노쉬 슬림쉐이크 더블초코`
- `단백하니 단백질쉐이크 시그니처`

제외 권장:

- `더단백 단백질쉐이크 파우더 곡물`
- `더단백 단백질쉐이크 파우더 초코`

이유:

- 파일명 기준으로 `파우더`가 명시되어 있어 현재 쉐이크 카테고리 정의와 맞지 않음

## 카드 템플릿

쉐이크 제품카드는 기존 음료 카드 구조를 유지하되, 아래 4개 지표를 기본으로 사용합니다.

- 단백질: `proteinPerServing`
- 칼로리: `calories`
- 당류: `sugar`
- 단백질 밀도: `density`

권장 보조 배지:

- `파우치형`
- `식사대용`
- `저당`
- `고단백`
- `식물성`

## 상세 상단 요약 템플릿

상세페이지 상단 요약은 아래 8개 항목을 기본으로 잡는 것이 적절합니다.

- 단백질
- 칼로리
- 당류
- 단백질 밀도
- 용량
- 제품 유형: `파우치형 쉐이크`
- 지방
- 나트륨

추가 fact chip 권장:

- 단백질원
- 식사대용 여부
- 식물성 여부
- 락토프리 여부

## 상세 영양성분표 템플릿

쉐이크는 `1팩 기준`으로 아래 순서로 통일합니다.

1. 칼로리
2. 탄수화물
3. 당류
4. 단백질
5. 지방
6. 포화지방
7. 트랜스지방
8. 콜레스테롤
9. 나트륨
10. 식이섬유
11. BCAA

운영 원칙:

- 라벨에 없는 값은 추정하지 않음
- 없는 항목은 `-`로 표기
- 단위는 `kcal / g / mg` 고정
- `100g당`만 있는 경우 총내용량 기준 환산 여부를 별도 확인 후 입력

## 데이터 템플릿

```ts
{
  slug: "sample-shake-choco-50",
  brand: "브랜드명",
  name: "제품명",
  capacity: "50g",
  flavor: "초코",
  variant: "파우치형",
  tags: ["파우치형", "식사대용"],
  proteinPerServing: 20,
  calories: 210,
  sugar: 5,
  fat: 7,
  sodium: 240,
  density: "9.5g/100kcal",
  calorieDensity: "0.10g/kcal",
  proteinSource: "분리대두단백",
  gradeTags: [],
  nutritionPerBottle: {
    caloriesKcal: 210,
    proteinG: 20,
    carbsG: 18,
    sugarsG: 5,
    fatG: 7,
    satFatG: 2.1,
    transFatG: 0,
    cholesterolMg: 0,
    sodiumMg: 240,
    fiberG: 6,
    bcaaMg: 0,
  },
}
```

## 확인이 필요한 항목

쉐이크는 제품별 라벨 형식 차이가 커서 아래는 실제 스펙 이미지 확인 후 입력해야 합니다.

- 총내용량 기준인지 `100g당` 기준인지
- 식사대용 성격인지 일반 단백질 보충형인지
- 파우더 포함 제품인지
- 식이섬유, BCAA, 콜레스테롤 표기 유무
- 물만 타먹는 파우더형인지, 바로 섭취 가능한 파우치형인지
